# Reference sections are conventions built from content paths and metadata, not first-class categories.

- Stego does not infer canonical category objects from the filesystem anymore.
- Teams can still organize reference leaves under paths such as `content/reference/concepts/` or group them with metadata such as `kind: reference`.
- Templates and the extension can project those leaves into sections, explorers, and appendices without a separate spine model.
